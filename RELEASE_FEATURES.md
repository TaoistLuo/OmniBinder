# OmniBinder Release Features

OmniBinder 是一个面向嵌入式 Linux、多进程系统以及分布式多板系统的分布式 IPC/RPC 框架。

产品为复杂业务系统提供统一的服务通信能力，支持本机高性能通信与跨设备网络通信，适用于传感器、控制、算法、网关、HMI、工业控制等场景。

---

## 产品概述

OmniBinder 提供统一的服务化通信模型，用于连接同一设备内的多个服务进程，以及跨板、跨设备的分布式服务节点。

通过统一运行时、服务注册发现机制、IDL 工具链与命令行工具，开发者可以使用一致的方式构建服务端、客户端和业务接口，而不需要分别处理本地 IPC 与跨设备 RPC 的差异。

---

## 核心特性

### 1. 分布式服务调用

支持以下调用模式：

- 同步 RPC
- one-way 调用
- 服务注册与发现
- 接口元数据查询

调用方无需区分本机与远端路径，可通过统一接口完成服务发现与远程调用。

### 2. 发布 / 订阅广播

支持 topic 模式的数据广播与订阅，适用于：

- 状态广播
- 实时数据推送
- 界面刷新
- 多模块状态同步
- 监控数据分发

发布者与订阅者通过统一机制建立关系，数据面走高效通信路径。

### 3. 服务死亡通知

支持服务死亡通知，用于感知目标服务在线状态变化。

该能力适用于：

- HMI / 上位机
- 控制器
- 网关
- 多服务协作系统

当目标服务退出或失效时，订阅方可收到统一通知，用于更新业务状态、进入离线模式或触发恢复流程。

---

## 混合传输模型

OmniBinder 采用混合传输模型：

- **本机优先 SHM**：同一设备内优先使用共享内存，以获得更低延迟和更高吞吐
- **跨机 TCP**：在跨设备、跨板场景下使用 TCP 建立统一调用路径

这一模型使业务层可以使用统一的调用方式，而底层根据部署关系自动选择更合适的传输路径。

---

## 控制面与数据面分离架构

OmniBinder 采用控制面与数据面分离架构：

### 控制面

由 ServiceManager 负责：

- 服务注册
- 服务发现
- topic 关系维护
- 服务死亡通知管理

### 数据面

负责：

- RPC 请求 / 响应
- topic 数据传输
- SHM / TCP 具体承载

这一架构既保证了统一管理，又避免了所有业务数据都绕经中心节点带来的性能限制。

## 开发与集成能力

### 统一运行时接口

产品提供统一运行时接口 `OmniRuntime`，支持：

- runtime 初始化
- 服务注册
- 服务发现
- RPC 调用
- topic 订阅与发布
- 死亡通知监听

对外提供线程安全公共 API，便于接入多线程业务系统。

### IDL 与代码生成

产品支持通过 `.bidl` 定义接口与数据结构，并通过 `omni-idlc` 自动生成 C / C++ 接入代码。

支持的 IDL 要素包括：

- `package`
- `import`
- `struct`
- `service`
- `topic`
- `publishes`

通过代码生成可自动获得：

- C++ Stub / Proxy
- C Stub / Proxy
- 数据结构编解码能力

### C / C++ 双栈支持

产品同时支持：

- 原生 C++ 集成
- 纯 C 集成

适合现代 C++ 业务系统，也适合现有 C 项目与嵌入式控制组件接入。

---

## 工具链支持

### omni-idlc

用于将 `.bidl` 接口定义转换为可直接接入业务工程的 C / C++ 代码。

### omni-cli

提供命令行调试与查询能力，支持：

- 服务列表查询
- 服务详情查询
- 接口信息查询
- 远程方法调用

适合作为联调、验证和部署阶段的辅助工具。

---

## 稳定性与恢复能力

产品当前已具备以下能力：

- ServiceManager 基础重连路径
- 控制面恢复主链路
- 长稳运行验证链路
- 断连恢复验证链路

这使产品能够支持持续集成、联调验证和业务系统接入。

## 下游独立集成示例

产品提供独立下游示例：

- `examples/artifact_sensor_hmi/`

该示例展示了如何在独立业务工程中：

- 使用安装产物接入 OmniBinder
- 通过 `find_package(OmniBinder REQUIRED)` 进行链接
- 使用 `omni-idlc` 进行接口代码生成
- 实现服务端与客户端
- 完成 RPC、pub/sub、死亡通知接入

示例同时提供：

- C++ 版本
- C 版本

适合作为客户接入与业务工程落地参考。

## 产品价值

OmniBinder 为客户提供：

- 统一的分布式 IPC/RPC 通信模型
- 面向本机与跨设备场景的一致调用体验
- 基于 SHM 的本机高性能通信能力
- 基于 TCP 的跨设备统一调用能力
- 可通过 IDL 驱动开发的接口定义方式
- 面向 C / C++ 的双栈接入能力
- 可直接参考的下游业务集成示例

这使其既可以作为通信基础设施，也可以作为业务系统快速集成与持续演进的基础平台。
