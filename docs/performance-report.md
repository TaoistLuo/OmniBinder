## 性能分析

### SHM eventfd 事件驱动路径

当前 SHM 传输已使用 eventfd + EventLoop 的事件驱动路径。
客户端写入请求后会触发服务端 req_eventfd，服务端写入响应后会触发客户端 resp_eventfd。
这意味着当前延迟数据主要反映：序列化、共享内存拷贝、eventfd 唤醒、epoll 调度与业务处理开销。

### 默认 SHM 容量说明

项目默认 SHM 配置为 `4KB / 4KB`，但本性能测试中的 `PerfService` / `PerfTopicService` 为了覆盖
4096-byte 与 8192-byte payload，显式使用了更大的服务级 SHM 配置（`16KB / 16KB`）。
因此本报告中的 4096-byte / 8192-byte 用例数据反映的是“显式放大 ring 后”的性能，而不是默认配置下的行为。

### 结论

- 0~1024 byte 常见 RPC 在当前机器上中位数约为 63~84 us
- 4096 byte payload 在当前测试配置下中位数约为 69 us
- topic pub/sub 在当前机器上中位数约为 38~53 us
